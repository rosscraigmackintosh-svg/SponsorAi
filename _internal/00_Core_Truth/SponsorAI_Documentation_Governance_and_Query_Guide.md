## 1. SponsorAI — Documentation Governance & Query Guide

Version: v1.0 

Last updated: 2026-02-23 

Authority: Highest (documentation integrity)

---

1. Purpose

Defines how the SponsorAI documentation corpus is:

- structured 

- queried 

- updated 

- versioned 

- extended 

Primary goal:

&gt; Ensure the SponsorAI knowledge base remains coherent, trustworthy, and usable as the system evolves.

This document governs the **docs as a system**, not the product itself.

---

2. Documentation Philosophy

SponsorAI documentation is treated as:

- living but controlled 

- extensible but structured 

- opinionated but auditable 

Hard rule:

&gt; Documentation clarity is a product asset, not an afterthought.

---

3. Authority Hierarchy

When conflicts arise, documents resolve in this order:

1. Constitution 

2. Ethical &amp; Regulatory Posture 

3. Commercial Guardrails 

4. Core Data &amp; Scoring Specs 

5. Product Definition Docs 

6. UX / IA Docs 

7. Working Memory 

8. Experiments / Archive 

AI systems and humans must follow this precedence.

---

4. Canonical Folder Roles

00_Core_Truth

Immutable foundations.

Contains:

- Constitution 

- Ethical posture 

- documentation governance 

- core doctrine 

Changes here require highest scrutiny.

---

01_Working_Memory

Current state snapshot.

Purpose:

- recent decisions 

- evolving reality 

- short-to-medium horizon truth 

Updated frequently.

---

02_Product_Definition

Strategic product intent.

Contains:

- positioning 

- opportunities rules 

- personalisation policy 

- commercial model 

Medium change frequency.

---

03_Design_System

Presentation and interaction rules.

Contains:

- export rules 

- visual language 

- motion principles 

---

04_Data_Models

Analytical spine.

Contains:

- FanScore 

- FitScore 

- confidence framework 

- ingestion 

- taxonomy 

- drift monitoring 

High technical authority.

---

05_UX_IA

Experience structure.

Contains:

- Explore philosophy 

- Control Room spec 

- navigation logic 

---

06_Experiments

Low-authority sandbox.

Content here is:

- exploratory 

- non-binding 

- forward-looking 

Must not override core doctrine.

---

99_Archive

Historical material only.

Never treated as current truth.

---

5. Document Versioning Rules

Every major doc must include:

- version number 

- last updated date 

- authority level 

When updating:

- increment version 

- preserve intent clarity 

- avoid silent semantic drift 

Major logic changes should be logged in Working Memory.

---

6. Change Classification

Documentation updates fall into three classes.

6.1 Clarification

- wording improvements 

- examples 

- formatting 

Low risk.

---

6.2 Behavioural Change

- scoring adjustments 

- ordering logic 

- eligibility rules 

Requires review and Working Memory log.

---

6.3 Structural Change

- new signals 

- taxonomy changes 

- commercial model shifts 

- marketplace boundary changes 

Requires:

- Constitution check 

- cross-doc review 

- explicit version bump 

---

7. Query Guidance for AI Systems

AI systems using this corpus should:

- prioritise Core Truth 

- respect signal separation 

- maintain neutrality 

- surface uncertainty 

- avoid prescriptive language 

When conflicts appear, AI should:

&gt; defer upward in the authority hierarchy.

---

8. Query Guidance for Humans

When reviewing SponsorAI docs, teams should ask:

- Is this consistent with the Constitution? 

- Does this preserve signal separation? 

- Does this maintain neutrality? 

- Does this introduce optimisation creep? 

- Is the tone still intelligence-first? 

If any answer is unclear → escalate.

---

9. Drift Detection (Documentation Layer)

Warning signs of documentation drift:

- multiple definitions of the same concept 

- creeping recommendation language 

- unexplained taxonomy changes 

- monetisation leaking into ranking 

- Explore behaving like a leaderboard 

If detected → initiate doc review.

---

10. Extension Protocol

Before adding a new major document:

Confirm:

- it does not duplicate an existing authority 

- it fits the folder structure 

- it respects the authority hierarchy 

- it reinforces trust-first positioning 

When in doubt, prefer **fewer, stronger documents**.

---

11. Review Cadence

Recommended rhythm:

- Working Memory — continuous 

- Data models — quarterly review 

- Commercial guardrails — biannual 

- Constitution — rare and deliberate 

- Ethical posture — annual review 

---

12. Long-Term Intent

The SponsorAI corpus is designed to become:

- internally queryable 

- AI-readable 

- investor-legible 

- regulator-comprehensible 

- team-extensible 

Clarity, restraint, and consistency are strategic advantages.

---

End of Document
