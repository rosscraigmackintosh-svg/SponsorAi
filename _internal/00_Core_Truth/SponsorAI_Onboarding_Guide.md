## 1. SponsorAI — New Team & Partner Onboarding Guide

Version: v1.0 

Last updated: 2026-02-23 

Authority: High (orientation &amp; adoption)

---

1. Purpose

Provides a structured entry point for new team members, partners, and collaborators to quickly understand:

- what SponsorAI is 

- how the system is structured 

- how to navigate the documentation 

- where to contribute safely 

Primary goal:

&gt; Reduce ramp time while preserving doctrinal consistency.

---

2. What SponsorAI Is (and Is Not)

SponsorAI is:

- a sponsorship intelligence platform 

- a market visibility layer 

- a decision-support surface 

SponsorAI is NOT:

- a marketplace 

- a broker 

- an ROI predictor 

- a pay-to-rank platform 

- a hype analytics dashboard 

If any feature begins to resemble the “NOT” list, it must be reviewed.

---

3. The Three-Signal Model (Foundational)

Every team member must understand the signal separation model.

FanScore → Attention 

Measures digital audience gravity.

FitScore → Alignment 

Measures contextual compatibility.

Budget / Minimum → Access 

Represents financial feasibility only.

**Hard rule**

&gt; These signals must never be collapsed into a single master score.

---

4. How the Product Is Structured

At a high level:

- Explore → structured discovery 

- Property Page → entity detail 

- Watchlist → lightweight tracking 

- Compare → side-by-side evaluation 

- Portfolio → active sponsorship set 

- Opportunities → declared openings 

New features must map cleanly into this mental model.

---

5. Documentation Map (How to Read the Corpus)

Recommended reading order for new team members:

1. Constitution 

2. Ethical &amp; Regulatory Posture 

3. Positioning Narrative 

4. FanScore Technical Spec 

5. FitScore Technical Spec 

6. Explore Ranking Philosophy 

7. Commercial Guardrails 

8. Working Memory (current state) 

This builds correct mental scaffolding.

---

6. Authority Hierarchy (Critical)

When documents conflict, precedence is:

1. Constitution 

2. Ethical &amp; Regulatory Posture 

3. Commercial Guardrails 

4. Data Model Specs 

5. Product Definition 

6. UX / IA 

7. Working Memory 

8. Experiments 

Always resolve upward.

---

7. How to Propose Changes

Before proposing a product or model change, ask:

- Does this preserve signal separation? 

- Does this maintain neutrality? 

- Does this introduce optimisation creep? 

- Could this be perceived as pay-to-play? 

- Is the output still explainable? 

If any answer is unclear → escalate.

---

8. Safe Contribution Areas

Lower-risk areas for iteration:

- UI clarity 

- filtering UX 

- export formatting 

- taxonomy extensions (with review) 

- opportunity freshness improvements 

Higher-risk areas requiring review:

- scoring logic 

- Explore ordering 

- commercial surfaces 

- personalisation behaviour 

- property permissions 

---

9. Common Failure Modes to Avoid

Teams working in this space often drift into:

- leaderboard energy 

- over-personalised feeds 

- hidden promotion mechanics 

- marketplace creep 

- false precision in analytics 

SponsorAI explicitly resists these patterns.

---

10. Tone &amp; Language Expectations

SponsorAI communication should remain:

- calm 

- grounded 

- analytical 

- neutral 

- confidence-weighted 

Avoid hype, certainty theatre, or betting-style language.

---

11. First 30-Day Success for New Team Members

A new contributor is successful when they:

- understand the three-signal model 

- can explain why SponsorAI is not a marketplace 

- can navigate the doc hierarchy 

- can identify trust risks in new ideas 

- can articulate FanScore vs FitScore correctly 

---

12. Long-Term Cultural Principle

SponsorAI is intentionally designed with restraint.

The product wins by being:

- clearer 

- calmer 

- more honest 

- more structurally trustworthy 

Not louder.
