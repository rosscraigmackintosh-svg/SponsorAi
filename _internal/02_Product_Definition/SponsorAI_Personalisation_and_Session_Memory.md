## 1. SponsorAI — Personalisation & Session Memory Policy

Version: v1.0 

Last updated: 2026-02-23 

Authority: High (experience integrity &amp; trust)

---

1. Purpose

Defines how SponsorAI may use user context, session behaviour, and saved preferences to personalise the experience.

Primary goal:

&gt; Improve discovery relevance while preserving neutrality, transparency, and user trust.

SponsorAI personalisation must remain **assistive, bounded, and explainable**.

---

2. Core Philosophy

SponsorAI does not operate an opaque optimisation feed.

Personalisation must:

- support exploration 

- preserve user agency 

- remain interpretable 

- avoid behavioural manipulation 

- avoid hidden ranking bias 

Hard rule:

&gt; Personalisation may guide the lens, but must not silently decide the outcome.

---

3. Types of Personal Context

SponsorAI recognises three distinct context layers.

3.1 Explicit User Signals

User-controlled actions such as:

- Watchlist additions 

- Compare selections 

- Portfolio entries 

- filter preferences 

- stated budget 

These are the **highest-trust signals**.

---

3.2 Session Context

Short-term behavioural context such as:

- current filters 

- recent views 

- active exploration area 

Session context is:

- temporary 

- reversible 

- non-authoritative 

---

3.3 Inferred Patterns (Future, tightly constrained)

May include:

- repeated sport focus 

- geography patterns 

- category interest signals 

These must remain:

- lightweight 

- explainable 

- non-dominant 

---

4. Permitted Personalisation Surfaces

SponsorAI may personalise:

Allowed

- “Related to your Watchlist” row 

- contextual FitScore reranking within filtered views 

- saved filter defaults 

- portfolio-aware comparisons 

- geography preselection 

These improve usability without distorting market visibility.

---

5. Prohibited Personalisation Behaviours

SponsorAI must NOT:

- create a fully personalised Explore feed 

- hide large parts of the market silently 

- reorder the entire landscape per user 

- create reinforcement loops 

- imply behavioural targeting 

- optimise for engagement time 

Hard rule:

&gt; Explore remains primarily market-led, not user-led.

---

6. Explore Page Guardrail

The Explore surface is intentionally **semi-stable and market-first**.

Therefore:

- core rows remain deterministic 

- global ordering remains data-led 

- personalisation may add context, not replace structure 

Acceptable:

- additional personalised rows 

- light contextual reordering inside filters 

Not acceptable:

- fully reshuffled home feed per user.

---

7. FitScore Personalisation Boundary

FitScore may incorporate:

- user geography 

- user category 

- user budget 

But must not:

- override FanScore visibility 

- create a single “best match” outcome 

- hide high-attention properties entirely 

FitScore is a **lens**, not a filter wall.

---

8. Transparency Expectations

Where meaningful, the system should allow users to understand:

- why something is being surfaced 

- what context is being applied 

- how FitScore is influencing ordering 

This may be achieved through:

- tooltips 

- subtle labels 

- contextual notes 

Over-explanation in the UI is not required, but explainability must exist.

---

9. Cold Start Behaviour

For new users with no signals:

System should default to:

- market-led Explore 

- neutral FitScore baseline 

- broad discovery 

SponsorAI must not fabricate personal assumptions.

---

10. Memory Retention Principles

User memory must be:

- user-controlled 

- revocable 

- bounded 

- purpose-limited 

The system must avoid building opaque long-term behavioural profiles in early versions.

---

11. Future ML Guardrails

If machine learning personalisation is introduced:

It must remain:

- interpretable 

- auditable 

- overrideable 

- confidence-aware 

Black-box recommendation engines are prohibited without explicit constitutional review.

---

12. Trust Risk Triggers

If users begin to feel:

- over-targeted 

- manipulated 

- trapped in a narrow feed 

- unable to rediscover the market 

…the personalisation model must be re-evaluated.

---

13. Success Criteria

Healthy personalisation should produce:

- faster discovery 

- better contextual relevance 

- preserved market visibility 

- stable user trust 

If engagement rises but trust signals fall, the system is miscalibrated.
